# QuickCalendar — Obsidian Plugin

## Project Overview
A year-at-a-glance calendar plugin for Obsidian that renders full-year calendars
inside notes via fenced code blocks. Inspired by the Calendar plugin by Liam Cain,
but designed for year-level views with multiple layout modes.

## Architecture

### Code Block Approach
The plugin registers a markdown code block processor for `quickcalendar` blocks.
Users write JSON configuration inside the block to control rendering:

```
\`\`\`quickcalendar
{
  "year": 2026,
  "view": "grid",
  "startDay": "monday",
  "weekNumbers": true
}
\`\`\`
```

### Source Structure
```
src/
├── main.ts              # Plugin entry point, code block processor registration
├── types.ts             # All TypeScript types, interfaces, and defaults
├── settings.ts          # Plugin settings tab UI
├── renderers/
│   ├── base-renderer.ts # Abstract base class with shared cell rendering
│   ├── grid-renderer.ts # 3×4 mini-calendar grid (timeanddate.com style)
│   ├── row-renderer.ts  # 12 horizontal month strips
│   ├── stream-renderer.ts # Continuous 7-column weekly flow
│   └── index.ts         # Renderer registry (add new views here)
└── utils/
    ├── date-utils.ts    # Date math, formatting, week calculations
    └── daily-notes.ts   # Daily note detection, creation, template support
```

### Adding a New View
1. Add the view name to `CalendarView` type in `src/types.ts`
2. Create `src/renderers/yourview-renderer.ts` extending `BaseRenderer`
3. Register it in `src/renderers/index.ts` RENDERERS map
4. Add CSS styles in `styles.css` under a `.qc-yourview-*` namespace

### Key Design Decisions
- **No external dependencies** for daily notes — we implement detection directly
  rather than depending on obsidian-daily-notes-interface (unmaintained)
- **Auto-detects** daily note settings from core Daily Notes and Periodic Notes plugins
- **Theme-aware CSS** — uses Obsidian CSS variables exclusively, no hardcoded colors
- **Extensible renderer pattern** — BaseRenderer provides shared cell/week rendering;
  subclasses only implement layout logic

## Build Commands
```bash
npm install          # Install dependencies
npm run dev          # Watch mode (esbuild)
npm run build        # Production build (type-check + bundle)
```

## Output Files
- `main.js` — bundled plugin (gitignored, generated by esbuild)
- `styles.css` — plugin styles (committed, loaded by Obsidian)
- `manifest.json` — plugin manifest (committed, required by Obsidian)

## Testing in Obsidian
1. `npm run build`
2. Copy `main.js`, `styles.css`, and `manifest.json` to your vault's
   `.obsidian/plugins/quickcalendar/` folder
3. Enable the plugin in Settings → Community Plugins
4. Create a note with a `quickcalendar` code block

## Daily Note Integration
- Clicking any date opens or creates the corresponding daily note
- Settings auto-detected from core Daily Notes / Periodic Notes plugins
- Can be overridden in plugin settings
- Template variables supported: `{{date}}`, `{{title}}`, `{{time}}`

## Extras
- `extras/Daily Note Template.md` — Ready-to-use Templater folder template (v20)
  that auto-applies daily note formatting when any `YYYY-MM-DD` named file is
  created in the Daily Notes folder. Works with wikilink clicks from QuickCalendar
  or anywhere in the vault.
- `extras/templater-daily-note-trigger.md` — Documentation explaining the folder
  trigger setup, how the date guard works, and the v20 migration from legacy
  `{{date:...}}` syntax to Templater `moment()` calls.

### Daily Note Template History
- The vault's canonical template is `templater/Daily Note - ACTIVE.md` in NovisVault
- v19 → v20: Replaced `{{date:MMMM Do, YYYY}}` and `{{date:dddd}}` (Obsidian core
  Templates syntax) with `moment(tp.file.title, "YYYY-MM-DD").format(...)` calls.
  The core syntax only works when the Daily Notes plugin creates the file; the
  Templater `moment()` equivalents work in all contexts including folder triggers.
- The folder trigger version (`extras/Daily Note Template.md`) is identical to
  `Daily Note - ACTIVE.md` v20 plus a date-format guard at the top.
